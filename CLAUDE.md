## Git Workflow

**IMPORTANT:** After every piece of work — whether it's a new feature, a bug fix, a refactor, or even a config change — you MUST commit and push to GitHub. Never leave work uncommitted. This ensures we never lose progress and can always revert to a previous state.

- Remote: https://github.com/NilsGit123/Claude-Code-Test
- Branch: `main`
- Commit after each logical unit of work (don't batch unrelated changes)
- Write clear, descriptive commit messages that explain *why* the change was made
- Push to `origin/main` immediately after committing
- When in doubt, commit more often rather than less
